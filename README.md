
## В проекте реализованы автоматизированные UI-тесты для приложения Pet на основе pytest и Selenium WebDriver.

### Структура тестов:

`test_profile_page.py` – тесты для страницы профиля  
`test_main_page.py` – тесты для главной страницы  
`test_login_page.py` – тесты для страницы логина (тесты, реализованные в прошлой итерации)

### Маркеры:
`@pytest.mark.smoke` – быстрые проверки наиболее критичной функциональности  
`@pytest.mark.regression` – расширенные регрессионные проверки основной бизнес-логики  
`@pytest.mark.skip` – кейсы реализованные в прошлой итерации

### Запуск тестов по маркерам:
`python -m pytest -m "smoke or regression" -v` – запуск всех тестов с маркерами *smoke* или *regression* в расширенном (`-v`) формате вывода  
`python -m pytest -v -s -m smoke` – запуск только *smoke*‑тестов с подробным выводом (`-v`) и показом вывода `print` (`-s`)
`python -m pytest -v -s -m regression` – запуск только *regression*‑тестов с подробным выводом и показом вывода `print`(`-s`)

#### Тесты страницы профиля (test_profile_page.py)
1. Добавление нового питомца только с обязательными полями [SMOKE]  
   _Проверяет, что новый питомец (pet1) создаётся при заполнении только обязательных полей._
   
3. Добавление нового питомца с обязательными и необязательными полями [REGRESSION]  
   _Проверяет, что новый питомец (pet2) создаётся при заполнении и обязательных, и необязательных полей._

4. Работа кнопки Cancel на странице создания питомца [SMOKE]  
   _Проверяет, что при нажатии кнопки Cancel изменения отменяются и новый питомец не создаётся._

5. Редактирование категории питомца [SMOKE]  
   _Проверяет, что категорию питомца можно изменить с cat на hamster._

6. Удаление питомца из списка [SMOKE]  
   _Проверяет, что питомца можно удалить из списка на странице профиля._

#### Тесты главной страницы (test_main_page.py)
6. Фильтрация по типу (type = hamster) под авторизованным пользователем [REGRESSION]  
   _Проверяет, что фильтрация по типу питомца hamster корректно работает для авторизованного пользователя._

7. Фильтрация по имени питомца под авторизованным пользователем [REGRESSION]  
   _Проверяет, что фильтрация по имени питомца корректно работает для авторизованного пользователя._

8. Добавление лайка питомцу [SMOKE]  
   _Проверяет, что пользователь может поставить лайк карточке питомца._

9. Переход на Pet Details по кнопке Details [REGRESSION]  
   _Проверяет, что по клику на кнопку Details открывается страница Pet Details._

10. Переход на страницу логина по кнопке Quit [REGRESSION]  
   _Проверяет, что по клику на кнопку Quit пользователь выходит из системы и перенаправляется на страницу логина._
